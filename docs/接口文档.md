# 接口文档

## 响应总结构（统一）
所有核心业务接口（`/api/analyze`、`/api/reviews`）统一返回：
- `status`: `success|error`
- `message`: 描述信息
- `data`: 当前操作结果
- `snapshot`: 后端权威快照

`snapshot` 结构：
- `cases`: 最新案件列表（权威状态）
- `disputes`: 最新纠纷列表
- `report_actions`: 报告中心动作日志（按 `case_id` 过滤）
- `workbench_refresh`: 工作台统计刷新字段
- `workbench`: 工作台渲染数据
- `risk`: 风险中心渲染数据

## 基础地址
- `GET /api/health`
- `GET|POST /api/samples`
- `GET|POST /api/reviews`
- `POST /api/analyze`

## `POST /api/analyze`
请求支持 `application/json` 或 `multipart/form-data`。

### 请求字段
- `geometry`：必填。ArcGIS `rings` 或 GeoJSON Polygon
- `area_mu`：可选
- `crop_mode`：可选，`wheat|corn`
- `model_mode`：可选，`rule|ml`
- `case_id`：可选，案件编号
- `operator`：可选，操作人
- `file`：可选，multipart 时传无人机影像

### 成功响应 `data`
- `analysis`：分析结果原始结构（`final_result/block_cells/...`）
- `case` / `latest_case`：最新案件对象
- `report_action`：本次业务动作（如分析完成）
- `workbench_refresh`：工作台刷新字段

### 失败响应
- `status=error`
- `message/error`
- `data.case`（若案件已创建）
- `snapshot`（用于前端统一 state replace）

## `GET /api/reviews`
可选参数：`case_id`（用于返回指定案件的报告动作日志）

### 成功响应 `data`
- `reviews`：复核记录列表
- `count`：复核记录数
- `latest_case`：最新案件对象
- `workbench_refresh`：工作台刷新字段

## `POST /api/reviews`
### 请求字段
- `case_id`：必填
- `reviewer`：可选
- `corrected_label`：可选
- `corrected_area_mu`：可选
- `comment`：可选
- `requires_resurvey`：可选

### 成功响应 `data`
- `review`：复核记录
- `count`：复核总数
- `case` / `latest_case`：完整最新案件对象
  - 至少包含：`status`、`reviewLabel`、`correctedAreaMu`、`reviewer`、`disputeStatus`、`lastReportAction`
- `review_outcome`：复核决策
- `dispute` / `dispute_status`：纠纷状态
- `report_action`：本次报告动作
- `workbench_refresh`：工作台统计刷新字段
