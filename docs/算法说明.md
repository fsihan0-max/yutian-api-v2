# 算法说明

## 分析算法（`algorithms/damage_analysis.py`）
- 保留既有遥感计算链路：
  - 非农地过滤
  - 像元分类（规则 + 简易样本模型）
  - 分块聚合（16x16）
  - 结果融合输出
- 由服务层统一编排，不再由接口直接转发 legacy 路由函数。

## 复核决策算法（`algorithms/review_decision.py`）
- `requires_resurvey=true`：退回重查
- 否则：复核结案

## 业务编排（服务层）
- `analysis_service.py`
  - 请求解析 -> 状态流转 -> 分析执行 -> 报告动作记录 -> 快照返回
- `review_service.py`
  - 复核入参校验 -> 复核落库 -> 纠纷更新 -> 案件更新 -> 报告动作记录 -> 快照返回

## 权威输出
`/api/analyze` 与 `/api/reviews` 统一返回：
- `status/message/data/snapshot`
- 前端据 `snapshot` 做统一 state replace，不再本地拼接“权威业务事件”。
