# 状态流转

## 权威原则
- 案件状态、纠纷状态、报告动作由后端维护并返回。
- 前端只做请求 + state replace + rerender，不直接本地改“已结案/纠纷已解决”。

## 状态定义
- `待查勘`
- `分析中`
- `待复核`
- `已结案`

## 转移规则
1. 分析发起：`待查勘|待复核|已结案 -> 分析中`
2. 分析成功：`分析中 -> 待复核`
3. 分析失败：`分析中 -> 待查勘`
4. 复核结案（`requires_resurvey=false`）：`待复核 -> 已结案`
5. 复核退回（`requires_resurvey=true`）：`待复核 -> 待查勘`

## 同步字段
每次分析/复核后，后端返回 `snapshot`，至少包含：
- `cases`
- `disputes`
- `report_actions`
- `workbench_refresh`

前端收到后统一替换状态并重绘页面。
